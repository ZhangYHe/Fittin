## ADDED Requirements

### Requirement: Shared Premium Screen Language
The system MUST apply a consistent premium visual language across the active workout, progress analytics, and profile/settings screens.

#### Scenario: User moves between primary tabs
- **WHEN** the user navigates between the home dashboard, workout flow, analytics, and profile destinations
- **THEN** the screens share the same spacing rhythm, surface treatment, section labeling, and typography hierarchy
- **AND** the non-home screens no longer appear visually disconnected from the home dashboard.

### Requirement: Shared Section and Surface Patterns
The system MUST render major screen regions using reusable soft-surface cards and grouped content sections instead of dense default layouts.

#### Scenario: User opens a redesigned non-home screen
- **WHEN** a screen contains multiple functional regions
- **THEN** each region is visually grouped using large-radius cards, subtle borders, and consistent padding
- **AND** section labels follow the same small-eyebrow grouping style used on the home dashboard.

### Requirement: Preserved Interaction Clarity
The system MUST improve visual consistency without reducing the clarity of primary interactions.

#### Scenario: User performs a core action
- **WHEN** the user logs a set, changes an analytics formula, or changes a setting
- **THEN** the relevant interactive element remains visually prominent, immediately discoverable, and easy to tap
- **AND** the redesign does not introduce extra interaction steps for those existing tasks.
